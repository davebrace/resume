CodeRequest.io

2019 - Present

I launched CodeRequest.io in 2019 to solve my own problem: managing take home projects while recruiting engineers. Prior to CodeRequest.io, I was manually creating private repositories containing a take home project on GitHub for each candidate. This was very cumbersome, but the benefits were clear as it allowed candidates to open pull requests on GitHub for us to review their take home projects. This let the candidate demonstrate their writing communication with code and also allowed them to learn about our code review culture.

vim-testnav

2016 - Present

I created a vim plugin, vim-testnav, in 2016 to enable rapid automatic switching between test and production code. The plugin is pretty simple, it will find a corresponding file name in the current project's hierarchy based on the file you're currently viewing. For example, if you're viewing 'home_controller_spec.rb', it will search for the 'home_controller.rb' file and open it for you.
